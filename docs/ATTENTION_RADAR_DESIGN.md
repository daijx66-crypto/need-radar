# 注意力雷达 v1 设计

日期：2026-07-11

## 目标

把现有“购买理由雷达”升级为每天可扫完的注意力保护型需求情报站：用户打开后先看到少量值得投入时间的变化，同时能继续追溯需求证据和低优先级背景。

核心承诺不是“收集更多”，而是回答四个问题：

1. 今天出现了什么新需求？
2. 哪些变化可能影响产品判断？
3. 哪些内容值得现在花时间？
4. 哪些内容可以放心忽略？

## 外部机制参考

### AI HOT

真源：<https://github.com/KKKKhazix/khazix-skills/tree/main/aihot>

只借机制：默认精选而非全量、滚动时间窗、日报归档、分类、关键词查询、多源热点和轻量新鲜度指纹。保留每条来源链接，不把摘要当原文。

### Follow Builders

真源：<https://github.com/zarazhangrui/follow-builders>

只借机制：追踪建造者而非搬运账号、来源白名单、没有新内容就跳过、每条必须有原始链接、摘要风格由本地配置控制、阅读记录留在本地。

该仓库 README 写明 MIT，但 GitHub API 未识别到独立许可证文件。本项目不复制其代码或文案，只采用通用产品机制。

## 产品结构

### 三条内容流

- `need`：高置信需求。来源于现有购买理由闸和八维评分。
- `shift`：重要变化。包括模型、产品、行业、论文、工具生态变化；它说明环境变化，不单独证明购买理由。
- `builder`：建造者信号。来自可信 builder、官方博客或深度播客；它说明值得关注的原始观点和实践。

第一版本地实现不新增外部付费源。`shift` 接入 AI HOT 的公开精选流，`builder` 接入 Follow Builders 的公开 JSON feed；两者仍写入统一 raw signal，因此不与本轮 UI 强耦合。现有 C 类趋势源也可进入 `shift`，但会先做热度归一化，不能挤占全部注意力名额。

### 注意力分层

每条内容保留原始分数，并新增独立的 `attention`：

- `now`：建议现在看，默认最多 5 条。
- `later`：值得稍后看，默认最多 8 条。
- `ignore`：可放心忽略，只显示数量和原因，需要时展开。

注意力分层必须有 `why_now` 和 `why_ignore`。热度只能作为变化证据之一，不能让泛热点进入需求主榜。

### 每日快照与变化

每次评分完成后保存 `data/history/YYYY-MM-DD.json`，并维护 `data/history/index.json`。跨日使用稳定条目 ID 计算：

- `is_new`
- `rank_delta`
- `score_delta`
- `streak_days`
- `evidence_delta`
- `change_label`

没有上一日快照时明确显示“首份快照”，不伪造升降。

## 个性化边界

`scorer/profile.json` 增加注意力偏好：关注主题、降低权重主题、优先来源、每日 now/later 上限。默认值保持通用，适合第一次打开的用户。

个性化只影响注意力排序与展示篇幅，不修改：

- 原始信号
- 购买理由闸标签
- 原始需求分
- 原始链接和证据

## 首页设计

首页从统计仪表盘改为内容优先：

1. 顶部显示数据新鲜度和一句话摘要。
2. 首屏显示“现在值得看”，按注意力优先级排列。
3. 导航切换“今日 / 新需求 / 重要变化 / 建造者 / 观察池”。
4. 每条显示类型、时间、来源、变化标签、为什么值得看和原始链接。
5. 现有八维需求详情继续保留。
6. 大量统计卡降权为紧凑摘要，观察池不再占据首页主体。

移动端同样优先显示 3 至 5 条 now 内容，不把桌面统计区机械堆叠成长页面。

## 错误与空态

- 数据超过 36 小时：显著提示“数据已过期”。
- 当日没有高置信需求：显示“今天没有新需求”，仍可展示重要变化。
- 没有历史：显示“首份快照”，隐藏升降数字。
- 外部来源不可用：采集器明确打印失败并写 0 条，整次刷新状态保留逐源结果；页面不拿历史内容伪装成今日新增。
- 缺原始链接：不进入 now/later。

## 验证

- 现有测试全部通过。
- 新增测试覆盖稳定 ID、首份快照、跨日新增/升降/持续、注意力上限和个性化不改原始分。
- `node --check web/app.js` 通过。
- 本地页面桌面与 390px 移动端无横向溢出、无 console error。
- 浏览器可验证今日分层、导航切换、需求详情和原始证据链接。

## 本轮边界

不启用 launchd/cron，不外发消息，不部署，不接入付费 API，不复制外部仓库代码，不重写现有采集器。完成本地数据契约、快照差异、注意力分层和首页第一版即可停止。
